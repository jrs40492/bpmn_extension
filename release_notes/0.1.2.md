## What's Changed

### BPMN:

- Add User Task properties panel with Task Configuration (TaskName, Actors, Groups, CreatedBy, Skippable, Priority) and Data I/O Mappings support
- Add Default Value and Process Variable binding to User Task form fields
  - New "Default Value" text field for hardcoded defaults, rendered as `value` attributes on generated form inputs
  - New "Process Variable" dropdown to bind a form field to a process-level variable for data reading in `setFormData()`
- Fix generated form JavaScript syntax errors when field names contain spaces or special characters (now uses bracket notation)
- Fix sequence flow arrows passing through center of nodes when opening BPMN files created in external editors (e.g., KIE/jBPM)
- Default new BPMN processes to `isExecutable="true"` to avoid compliance warnings
- Fix gateway coloring not applying to BPMN files with non-standard element IDs (e.g., UUID-style IDs from external editors)
- Fix process variables not displaying for BPMN files created in IBM BAMOE (which stores variables as standard `bpmn:Property` elements rather than custom extension elements)
- Fix generated forms not using a process variable's default value when a form field binds to it via the "Process Variable" dropdown
- Add complex JSON rendering for generated User Task forms: object-typed fields with JSON default values now render as expanded fieldsets with individual inputs, and arrays of objects render as editable card lists with add/remove buttons
- Fix process variable deletion not cleaning up user task `dataInput` references, causing stale `drools:dtype` and `itemSubjectRef` that lead to Kogito compilation errors
- Fix process variable type changes not propagating to user task `dataInput` elements (`drools:dtype` now updates correctly)
- Fix `findProperty` failing on BPMNs imported from external editors (UUID-style property IDs are now matched by name)
- Fix `itemDefinition` lookup on delete using ID pattern instead of following `itemSubjectRef` reference, which failed on imported BPMNs
- Add "Custom" type option for process variables, allowing fully qualified Java class names (e.g., `com.cmm.pr.PullRequest`) to be preserved and edited instead of being collapsed to "Object"
- Fix Kogito persistence errors by mapping `object` type to `java.io.Serializable` instead of `java.lang.Object` (which doesn't implement `Serializable`)
